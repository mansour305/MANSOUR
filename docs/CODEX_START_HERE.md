# Codex Start Here

Start every new task by reading `AGENTS.md`.

Work in controlled tasks only. Keep each task narrow, explicit, and tied to the current branch and repository target.

Do not claim production readiness until a complete audit, security review, data-source review, QA pass, and deployment verification have been completed with evidence.

Do not burn usage limits with broad repeated scans. Read only the files needed for the current task, and avoid re-reading the entire repository unless the task is a full audit.

The first real task after this setup is a full project audit. That audit should establish the actual architecture, source of truth, auth/admin behavior, Supabase/RLS posture, API reliability, frontend UX state, deployment readiness, and open issue ledger.
