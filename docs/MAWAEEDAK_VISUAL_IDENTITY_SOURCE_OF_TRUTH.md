# مواعيدك — مصدر الحقيقة البصري الرسمي

## الحكم

الصور المرجعية المرفقة في محادثة اعتماد الهوية بتاريخ 2026-06-01 هي **Mandatory Visual Reference** وليست إلهاماً أو moodboard.

أي تنفيذ لاحق داخل تطبيق مواعيدك يجب أن يطابق هذه الهوية ولا يستبدلها بهوية جديدة.

## الشخصية البصرية

- سعودي فاخر وهادئ.
- رسمي، دافئ، منظم، واضح، ومريح للعين.
- Mobile-first RTL أولاً، ثم دعم لوحة المالك desktop/tablet بنفس اللغة البصرية.
- واجهات ناعمة بزوايا مستديرة وظلال خفيفة ومساحات بيضاء واسعة.
- عناصر زخرفية خفيفة فقط: نخيل، فوانيس، معمار سعودي معاصر، أقواس، زخارف رملية ناعمة.
- لا ازدحام، لا ألوان صارخة، لا ثيمات غامقة كهوية افتراضية.

## الخطوط

- خط الواجهات والنصوص: `Cairo` أو `Noto Kufi Arabic`.
- العناوين والشعار: خط عربي فاخر قريب من الديواني أو خط مخصص للشعار.
- الأرقام: Cairo واضح ومتوازن.
- يمنع استخدام خطوط تقنية باردة أو حادة لا تناسب الهوية.

## قواعد المكونات

### البطاقات

- خلفية: أبيض دافئ أو كريمي.
- Radius: من 18 إلى 28px حسب حجم البطاقة.
- Border: `1px solid rgba(138, 107, 61, 0.12)`.
- Shadow: خفيف جداً، لا يكون قاسياً.
- Padding واسع وواضح.

### الأزرار

- Primary: تدرج ذهبي/بني هادئ.
- Secondary: كريمي بحد ذهبي خفيف.
- Danger: أحمر ترابي للنص أو الحدود فقط، لا يستخدم كتلة صاخبة إلا للضرورة.

### الأيقونات

- أسلوب خطي ناعم.
- اللون الافتراضي: بني/ذهبي.
- لا تستخدم أيقونات متعددة الأنماط في نفس الصفحة.
- لا تستخدم أيقونات ثلاثية الأبعاد أو ملونة عشوائياً.

### الشريط السفلي

- Mobile-first RTL.
- عناصر رئيسية: الرئيسية، الرواتب/المواعيد حسب البنية، التقويم، الخدمات، المزيد.
- العنصر النشط بلون ذهبي/بني مع خلفية كريمية ناعمة.
- لا يظهر Admin أو أدوات مالك للمستخدم العادي.

### القائمة الجانبية

- مطابقة مرجع القائمة المرفق: بطاقة ترحيب، شعار، تاريخ، أزرار كبيرة مريحة، خلفية معمارية/نخيل ناعمة في الأسفل.
- العناصر الأساسية: الرئيسية، بطاقة يومية أرسلها، شارك التطبيق، سياسة الخصوصية، الشروط والأحكام، المساعدة والدعم، تسجيل الخروج.

## الشاشات الملزمة

### الصفحة الرئيسية

يجب أن تحتوي على:

- شعار مواعيدك في الأعلى.
- التاريخ الهجري والميلادي.
- بطاقة صباحية/رسالة يومية بصورة معمارية سعودية هادئة.
- مواقيت الصلاة ببطاقات ناعمة.
- رسالة دينية قصيرة عن الصلاة.
- عد تنازلي للصلاة القادمة.
- مواعيد مالية قريبة بعدادات واضحة.
- شريط سفلي مطابق للهوية.

### صفحة المزيد

يجب أن تحتوي على:

- بطاقة مستخدم فاخرة في الأعلى.
- عناصر: الملف الشخصي، الإعدادات، مشاركة التطبيق، قصة اليوم، سياسة الخصوصية، الشروط والأحكام، المساعدة والدعم، تسجيل الخروج.
- عبارة ختامية هادئة مثل: بارك الله في وقتك.

### صفحة الخدمات

يجب أن تكون Grid ثنائي البطاقات Mobile-first وتشمل:

- نظم مواعيدك.
- رحلاتي القادمة.
- الشكاوى والاقتراحات.
- قدم تهنئة لمن تريد.
- اتصل بنا.
- الأخبار.
- وظائف.

### لوحة المالك

- Desktop RTL بنفس الهوية.
- Sidebar يمين.
- Logo واضح.
- Cards إحصائية ناعمة.
- Charts بألوان الهوية فقط.
- لا تستخدم admin dashboard غامق أو أزرق تقني افتراضي.

## الممنوعات

- ممنوع اعتبار الصور إلهاماً فقط.
- ممنوع تغيير الشعار أو الألوان الأساسية دون موافقة المالك.
- ممنوع إدخال ثيمات مخالفة كهوية افتراضية.
- ممنوع استخدام عناصر حكومية رسمية مثل السيفين والنخلة كشعار للتطبيق.
- ممنوع ظهور Demo Mode أو Coming Soon أو أدوات مالك للمستخدم العادي.
- ممنوع أزرار ميتة أو صفحات فارغة.

## معيار القبول البصري

لا تعتبر أي مرحلة Visual Passed إلا إذا:

1. الصفحة الرئيسية تطابق الهوية في اللون، الخط، البطاقات، المسافات، والروح العامة.
2. القائمة الجانبية تطابق المرجع.
3. صفحة المزيد تطابق المرجع.
4. صفحة الخدمات تطابق المرجع.
5. لوحة المالك تطابق مرجع Admin المرفق.
6. لا يوجد ثيم افتراضي يخالف الهوية.
7. لا يوجد كسر RTL أو اختلاف جذري في النبرة البصرية.
